# 아키텍쳐 문서

## 개요

똑닥(DDokDak)은 Go 언어로 제작한 디스코드 봇으로, 사용자의 메시지를 받아 AI 응답을 생성하는 시스템입니다.
Antigravity CLI(agy)를 백엔드로 사용하여 다양한 AI 모델을 활용합니다.

## 패키지 설명

### `cmd/ddokdak` — 엔트리포인트
- 애플리케이션 시작점
- 설정 로드 → 봇 생성 → 시작 → 시그널 대기 → 종료

### `internal/config` — 설정 관리
- 환경변수에서 설정값 로드
- 필수값 검증 (DISCORD_TOKEN)
- 기본값 적용 (모델명, 타임아웃, 로그 레벨)
- `time.Duration` 파싱 및 로그 레벨 유효성 검사

### `internal/bot` — 봇 코어
- 디스코드 세션 생성 및 관리
- 이벤트 핸들러 등록
- 슬래시 커맨드 등록/해제
- 시그널 기반 우아한 종료

### `internal/command` — 슬래시 커맨드
**register.go:**
- `/채널지정` — 텍스트 채널을 AI 대화 채널로 등록
- `/채널해제` — AI 대화 채널 등록 해제
- `/모델변경` — 글로벌/세션별 AI 모델 동적 변경
- `/사용량` — 모델별 사용량 대시보드 (1분 자동 업데이트)
- 글로벌/길드 단위 커맨드 등록 지원

**handler.go:**
- 인터랙션 이벤트 처리
- 채널 등록/해제, 모델 변경, 사용량 대시보드 로직
- Ephemeral 응답 (명령자에게만 보이는 응답)

### `internal/handler` — 메시지 처리
- 메시지 생성 이벤트 감지
- 봇/자기 자신 메시지 필터링
- 지정된 채널 여부 확인
- 쓰레드 자동 생성 (AutoArchiveDuration: 7일)
- **순차 큐 처리**: AI 응답 생성 중 새 메시지는 MsgChan 버퍼에 쌍이고, 응답 완료 후 순차 처리
- **메시지 디바운싱**: 2초 대기 후 여러 메시지를 하나로 묶어 처리
- **첨부파일 처리**: 자동 다운로드 후 AI에게 파일 경로 전달
- 타이핑 인디케이터 표시
- 긴 응답 분할 전송 (2000자 제한)
- 사용량 추적기와 연동 (RecordCall/RecordError)

### `internal/session` — 세션 관리
**session.go:**
- UUID v4 기반 세션 식별자 (crypto/rand)
- 쓰레드 ID ↔ Antigravity CLI 대화 ID 매핑
- 스레드 안전한 ConversationID 접근

**manager.go:**
- 쓰레드 ID 기반 세션 CRUD
- `sync.RWMutex` 기반 동시성 제어

### `internal/store` — 채널 저장소
- `ChannelStore` 인터페이스 정의
- `InMemoryChannelStore` 인메모리 구현
- 길드별 채널 ID 관리 (`map[string]map[string]struct{}`)
- `sync.RWMutex` 기반 동시성 안전

### `internal/agy` — Antigravity CLI 클라이언트
- `exec.CommandContext`를 통한 agy CLI 호출
- 새 대화 / 기존 대화 이어가기 지원
- 동적 모델 선택 (세션/글로벌 모델)
- 쓰레드별 로그 파일 관리
- stdout 응답 파싱
- stderr 기반 에러 핸들링

### `internal/downloader` — 파일 다운로더
- 디스코드 첨부파일 HTTP 다운로드
- 타임스탬프 + 원본 파일명으로 안전한 파일명 생성
- 경로 순회 공격 방지 (sanitizeFilename)
- 절대 경로 반환

### `internal/usage` — 사용량 추적 및 대시보드
**tracker.go:**
- `Tracker`: 모델별 호출 횟수/오류 횟수/마지막 사용 시간 추적
- `RecordCall()`, `RecordError()`: 스레드 안전한 사용량 기록
- `GetUsages()`: 호출 횟수 내림차순 정렬
- `FormatDashboard()`: 이모지 포맷된 디스코드 메시지 생성 (KST 시간대)
- `Dashboard`: 1분마다 ChannelMessageEdit으로 자동 업데이트
- 우아한 종료 (stopChan)

## 데이터 흐름

### 새 대화 시작

```
1. 사용자가 지정된 채널에 메시지 전송
2. MessageHandler가 메시지 수신
3. 채널 등록 여부 확인 (ChannelStore)
4. 메시지 내용으로 쓰레드 생성 (AutoArchiveDuration: 7일)
5. 새 세션 생성 (SessionManager)
6. 비동기로 AI 응답 처리:
   a. 타이핑 인디케이터 표시
   b. agy CLI 호출 (새 대화)
   c. ConversationID 저장
   d. 응답 쓰레드에 전송
```

### 기존 대화 이어가기

```
1. 사용자가 쓰레드에 메시지 전송
2. MessageHandler가 쓰레드 메시지 감지
3. 세션 조회 (SessionManager)
4. 마지막 활동 시간 갱신
5. 비동기로 AI 응답 처리:
   a. 타이핑 인디케이터 표시
   b. agy CLI 호출 (--conversation 플래그로 기존 대화 이어가기)
   c. 응답 쓰레드에 전송
```

## 동시성 모델

- **채널 저장소**: `sync.RWMutex`로 읽기/쓰기 분리
- **세션 매니저**: `sync.RWMutex`로 세션 맵 보호
- **세션 객체**: `sync.Mutex`로 내부 상태 보호
- **AI 응답 처리**: 고루틴으로 비동기 처리 (메시지 수신 블로킹 방지)
- **타이핑 인디케이터**: 별도 고루틴에서 5초 간격 갱신

## 설계 원칙

1. **의존성 주입** — 모든 핸들러는 생성자를 통해 의존성을 주입받습니다.
2. **인터페이스 분리** — `ChannelStore` 인터페이스로 저장소 구현을 추상화합니다.
3. **internal 패키지** — 모든 핵심 로직은 `internal/` 하위에 배치하여 외부 접근을 방지합니다.
4. **동시성 안전** — 모든 공유 상태는 뮤텍스로 보호됩니다.
5. **테스트 가능성** — 핵심 로직은 단위 테스트가 가능하도록 설계되었습니다.
6. **우아한 종료** — SIGINT/SIGTERM 시그널을 처리하여 리소스를 정리합니다.

## 확장 가능성

- **영속 저장소**: `ChannelStore` 인터페이스를 구현하여 DB 기반 저장소로 교체 가능
- **모델 폴백**: `AgyFallbackModel` 설정으로 기본 모델 실패 시 대체 모델 사용 가능
- **다중 길드**: 길드별 채널 관리로 다중 서버 동시 지원
- **웹훅 통합**: 응답 포맷을 Embed로 확장하여 더 풍부한 UI 제공 가능
